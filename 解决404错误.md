# 🔧 解决 404 错误（main.tsx）

## 问题原因

GitHub Pages 可能还在使用旧版本的 HTML 文件，或者浏览器缓存了旧版本。

## ✅ 解决方案

### 方法 1：清除浏览器缓存（最简单）

1. **Chrome/Edge**：
   - 按 `Ctrl+Shift+Delete` (Windows) 或 `Cmd+Shift+Delete` (Mac)
   - 选择"缓存的图片和文件"
   - 点击"清除数据"
   - 或者按 `Ctrl+Shift+R` (Windows) 或 `Cmd+Shift+R` (Mac) 强制刷新

2. **Safari**：
   - 按 `Cmd+Option+E` 清除缓存
   - 或按 `Cmd+Shift+R` 强制刷新

3. **Firefox**：
   - 按 `Ctrl+Shift+Delete` (Windows) 或 `Cmd+Shift+Delete` (Mac)
   - 选择"缓存"
   - 点击"立即清除"

### 方法 2：检查 GitHub Actions 部署状态

1. 访问：https://github.com/ziqichen55555/thelittlenestcalendar/actions
2. 查看最新的部署工作流是否成功
3. 如果失败，查看错误信息
4. 如果成功，等待 1-2 分钟让 GitHub Pages 更新

### 方法 3：使用无痕/隐私模式

1. 打开浏览器的无痕/隐私模式
2. 访问：https://ziqichen55555.github.io/thelittlenestcalendar/
3. 如果无痕模式可以正常显示，说明是缓存问题

### 方法 4：检查部署的 HTML

访问：https://ziqichen55555.github.io/thelittlenestcalendar/

查看页面源代码（右键 → 查看页面源代码），应该看到：

```html
<script type="module" crossorigin src="/thelittlenestcalendar/assets/index-XXXXX.js"></script>
```

**不应该**看到：
```html
<script type="module" src="/src/main.tsx"></script>
```

## 🔍 调试步骤

1. 打开浏览器开发者工具（F12）
2. 切换到 **Network**（网络）标签
3. 勾选 **"Disable cache"**（禁用缓存）
4. 刷新页面（F5）
5. 查看哪些资源返回 404

## 📝 当前构建状态

- ✅ 本地构建文件正确
- ✅ `dist/index.html` 正确引用了构建后的文件
- ⚠️ GitHub Pages 可能还在使用旧版本

## 💡 如果还是不行

1. 等待 2-3 分钟让 GitHub Pages 完全更新
2. 检查 GitHub Actions 是否有错误
3. 尝试手动触发部署：
   - 访问 Actions 页面
   - 点击 "Deploy to GitHub Pages" 工作流
   - 点击 "Run workflow"
